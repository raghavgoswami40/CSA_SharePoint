# SharePoint SPFx Project

## Frontend Design Principles
(Based on anthropics/skills frontend-design)

### Design Philosophy
- **Hero as thesis**: Open with the most characteristic element
- **Typography as personality**: Pair display and body typefaces deliberately
- **Structure encodes meaning**: Use layout elements only when they communicate information
- **Motion serves purpose**: Animate deliberately or not at all
- **Spend boldness in one place**: Let one signature element carry the distinctive idea

### Process
Always work in two passes:
1. Brainstorm token system (color, typography, layout, signature element)
2. Critique that plan against the brief before building

### Guardrails
- Avoid generic defaults: cream/terracotta, near-black/acid-green, broadsheet layouts
- Maintain responsive design, keyboard accessibility, reduced-motion support
- Copy should clarify navigation, not decorate — use active voice and plain language

### Tech Stack
- SPFx with React
- Fluent UI components
- SASS modules for styling
- SharePoint Online target
